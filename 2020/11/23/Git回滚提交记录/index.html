2. 回滚本地提交记录

命令 影响暂存区 影响工作区
git reset –soft N N 执行此命令后，将当前分支的本地仓库回退到commId对应的那次提交，并将已提交的文件保存到暂存区。工作区和暂存区中的其他文件不受影响
git reset –mixed Y N 执行此命令后，将当前分支的本地仓库回退到commId对应的那次提交，将已提交的文件保存到工作区，暂存区中的其他文件也将被保存到工作区，即清空了暂存区。–mixed是默认参数，如果git reset 命令不加参数，将使用默认参数–mixed
git reset –hard Y Y 执行此命令后，将当前分支的本地仓库回退到commId对应的那次提交，已提交的文件会丢失，原暂存区和工作区会被清空

执行上述命令时，除了使用<commitId>以外，也可以使用head
