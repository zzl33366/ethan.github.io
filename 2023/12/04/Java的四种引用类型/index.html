Java的四种引用类型

2023-12-04
Word count: 725   |   Reading time≈ 2 min

Java有强、软、弱、虚四种引用，本文将依次介绍这4种引用。

简介

  1. 强引用(Strong Reference)

    Java中引用默认就是强引用，任何一个对象的赋值操作就产生了这个对象的强引用。强引用的特点是只要有强引用存在，被引用的对象就不会被回收

  2. 软引用(Soft Reference)

    软引用的类定义

    public class SoftReference extends Reference

    SoftReference有两种构造方法

    public SoftReference(T referent)

    public SoftReference(T referent, ReferenceQueue<? super T> q)

    软引用的特点是只有在内存不足的情况下，被引用的对象才会被回收

  3. 弱引用(Weak Reference)

    弱引用与软引用的构造方法类似，同样拥有一个参数的构造方法和两个参数的构造方法。弱引用的特点是只要垃圾回收机制执行，被弱引用的对象就会被回收，不论内存是否充足

  4. 虚引用(Phantom Reference)

    虚引用有且只有以下一个构造方法

    public PhantomReference(T referent, ReferenceQueue<? super T> q)

    虚引用必须和引用队列联合使用，“虚”顾名思义“形同虚设”，与其他几种引用不同，虚引用不会决定对象的生命周期，如果一个对象仅持有虚引用，那就和没有引用一样，在任何时候都可能被回收。通过虚引用不能获取到指向对象的实例。

    虚引用的主要作用是跟踪对象被垃圾回收的状态，在垃圾回收的过程中，如果发现还有虚引用指向待回收的内存区域时，会在回收内存之前，将此虚引用加入到引用队列中，程序可以通过判断引用队列中是否加入了虚引用（调用poll方法），来了解指向区域的内存是否将要被回收

Reference的生命周期

上一章节提到的引用构造函数有2种，分别是一个参数的构造方法和两个参数的构造方法，两个参数的构造方法中，第二个参数的类型为ReferenceQueue。根据上图，Reference的生命周期描述如下：

  • 无论是否带了ReferenceQueue实例化的Reference，状态都是Active
  • 当可达性发生变化，不带ReferenceQueue的Reference状态会变为Inactive。带ReferenceQueue的Reference状态会变为Pending，意为等待进入Queue，进入到Queue中的Reference状态变为Enqueued，调用poll方法将其从队列中取出后，此Reference的状态变为Inactive
  • 到达Inactive状态的Reference状态不能被改变，等待回收
